# エージェント依頼タスク分類

Cursorのエージェントに依頼する作業は、性質ごとに分けると扱いやすくなります。
分類できない依頼は、たいてい成果物か判断基準が曖昧です。

## 1. 調査・リサーチ

### 目的

意思決定の前に、必要な情報を集めて整理する。

### 依頼例

- 競合企業のWebサイト構成を比較する
- 既存キャンペーンの訴求軸を整理する
- GitHubリポジトリ内の関連ドキュメントを探す
- 過去のPRやIssueから類似事例を抽出する

### 成果物

- 調査メモ
- 比較表
- 要点サマリー
- 次に確認すべき論点

### 注意点

- 情報源を明記させる
- 推測と事実を分ける
- 最終判断は人間が行う

## 2. 文章作成・編集

### 目的

社内外向けの文章を作成、改善、再構成する。

### 依頼例

- プレスリリースの下書きを作る
- 社内告知文を短くする
- 営業メールのトーンを調整する
- FAQを顧客向けの表現に直す

### 成果物

- Markdown文書
- メール文面
- FAQ
- SNS投稿案

### 注意点

- 読者を指定する
- 口調や禁止表現を伝える
- 公開前に人間が事実確認する

## 3. 資料・講義コンテンツ作成

### 目的

説明会、研修、社内勉強会で使う材料を作る。

### 依頼例

- Cursor講義のアジェンダを作る
- ハンズオン課題を作る
- スライドの見出し案を作る
- 受講者向けチェックリストを作る

### 成果物

- 講義アウトライン
- ハンズオン手順
- スライド構成案
- チェックリスト

### 注意点

- 受講者の前提知識を指定する
- 演習の完了条件を明確にする
- 実データや機密情報を使わない

## 4. GitHub運用支援

### 目的

Issue、Pull Request、レビューを使った業務管理を補助する。

### 依頼例

- Issueテンプレートを作る
- PR説明文を改善する
- 変更内容をレビュー観点で要約する
- タスクをラベル別に分類する

### 成果物

- Issueテンプレート
- PRテンプレート
- レビューコメント案
- 運用ルール文書

### 注意点

- 書き込み権限の扱いを確認する
- 自動化する範囲を限定する
- 承認プロセスを明文化する

## 5. データ整理・分析補助

### 目的

表、CSV、ログ、アンケートなどを整理し、判断しやすい形にする。

### 依頼例

- アンケート回答をカテゴリ別に整理する
- CSVの列定義を説明する
- 顧客フィードバックをテーマ別に分類する
- キャンペーン結果の要点をまとめる

### 成果物

- 集計表
- 分類リスト
- インサイトメモ
- 次の施策候補

### 注意点

- 個人情報を扱う場合は匿名化する
- 数値の解釈を鵜呑みにしない
- 元データの前提を残す

## 6. 業務フロー改善

### 目的

繰り返し作業を整理し、手順化または自動化する。

### 依頼例

- 毎月のレポート作成手順を標準化する
- キャンペーン開始前チェックリストを作る
- Slack通知文のテンプレートを作る
- 承認フローを図解する

### 成果物

- 手順書
- チェックリスト
- テンプレート
- フロー図

### 注意点

- 現場で実行できる粒度にする
- 例外処理を明記する
- 責任者と確認者を分ける

## 7. コード・自動化支援

### 目的

軽量なスクリプトや設定ファイルを作り、作業を効率化する。

### 依頼例

- Markdownファイルのリンク切れを検査する
- CSVを整形するスクリプトを作る
- GitHub Actionsの設定を作る
- READMEの目次を自動生成する

### 成果物

- スクリプト
- 設定ファイル
- 実行手順
- テスト結果

### 注意点

- 実行環境を指定する
- 破壊的な操作を避ける
- テスト方法をセットで依頼する

## 依頼の基本形

```text
目的:
背景:
対象ファイルまたは対象情報:
成果物:
制約:
確認方法:
```

泣き言はいい。曖昧な依頼は曖昧な成果しか返さない。任務は具体化するんだ。
