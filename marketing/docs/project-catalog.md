# プロジェクト分類カタログ

Cursorエージェントに依頼しやすいプロジェクトを、社内講義で説明しやすい単位に分類します。

## 1. マーケティング基盤整備

### 目的

施策を実行する前に、情報・資料・データ・運用ルールを整理する。

### プロジェクト例

- LPやWebサイトの構成整理
- キャンペーン資料のテンプレート化
- メール配信文面の管理ルール作成
- CRMやスプレッドシート項目の棚卸し
- KPI定義書の作成

### エージェントへの依頼例

```text
既存のキャンペーン資料を読み取り、目的・ターゲット・訴求軸・CTAの観点で分類してください。
不足している情報があれば、GitHub Issueのチェックリストとして整理してください。
```

## 2. コンテンツ制作支援

### 目的

記事、SNS投稿、営業資料、FAQなどの制作速度と品質を上げる。

### プロジェクト例

- ブログ記事の構成案作成
- SNS投稿案の量産
- ホワイトペーパーの目次作成
- セミナー告知文の作成
- 顧客事例インタビューの質問案作成

### エージェントへの依頼例

```text
Cursor初心者向け社内講義の記事案を作成してください。
対象読者は非エンジニアです。導入、基本操作、依頼文の作り方、注意点、演習の順で構成してください。
```

## 3. 業務改善・自動化

### 目的

繰り返し作業や手作業の確認を減らし、担当者が判断に集中できる状態を作る。

### プロジェクト例

- 定型レポートの作成手順整理
- チェックリストの自動生成
- CSVやスプレッドシートの加工ルール作成
- 社内FAQの更新
- GitHub Issueテンプレートの整備

### エージェントへの依頼例

```text
毎週のマーケティングレポート作成手順を読みやすい手順書にしてください。
手作業、エージェントに任せる作業、人間が確認する作業に分けてください。
```

## 4. 営業・カスタマーサクセス支援

### 目的

顧客接点で使う資料や回答品質を標準化する。

### プロジェクト例

- 商談前リサーチ項目の整理
- 提案書テンプレート作成
- FAQ回答案の作成
- 導入事例の要約
- 顧客セグメント別トークスクリプト作成

### エージェントへの依頼例

```text
新規顧客向けの提案準備チェックリストを作成してください。
顧客理解、課題仮説、提案資料、確認事項、次アクションに分類してください。
```

## 5. 社内教育・ナレッジ共有

### 目的

Cursorの利用方法や成功事例を社内に広げる。

### プロジェクト例

- 社内講義資料の作成
- 利用ルールのドキュメント化
- よくある依頼文のテンプレート化
- 成功事例・失敗事例の整理
- 新入社員向けオンボーディング資料作成

### エージェントへの依頼例

```text
社内でCursorを使うときのルールを作成してください。
機密情報、レビュー責任、成果物確認、GitHub運用、禁止事項に分けてください。
```

## プロジェクト化の判断基準

| 判断項目 | タスクでよい | プロジェクト化する |
| --- | --- | --- |
| 成果物 | 1つの文書や修正 | 複数の成果物がある |
| 関係者 | 依頼者のみ | 複数部署が関わる |
| 確認方法 | 依頼者が確認できる | レビューや承認が必要 |
| 継続性 | 一度で終わる | 継続運用がある |
| 影響範囲 | 個人業務 | チーム運用 |

## 優先順位の付け方

1. 繰り返し発生している作業
2. 手順が曖昧で属人化している作業
3. 成果物の品質差が大きい作業
4. レビューに時間がかかっている作業
5. 社内展開すれば効果が広がる作業

戦場では補給線が命だ。会社なら、情報の整理がそれに当たる。派手な施策の前に、足場を固めるんだ。
