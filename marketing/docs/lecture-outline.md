# Cursor社内講義アウトライン

## 講義の目的

Cursorを単なるコードエディタではなく、業務タスクを整理し、AIエージェントへ依頼し、成果物をレビューするための作業環境として理解する。

## 到達目標

受講者が講義後にできるようになること:

- Cursorでファイル、チャット、エージェント作業の関係を説明できる
- エージェントに任せるタスクを適切な粒度に分けられる
- GitHub Issueとして依頼内容を整理できる
- Pull Requestでエージェントの成果物を確認できる
- 社内業務に使える小さな自動化テーマを見つけられる

## 講義構成

### 1. Cursorとは何か

- AI支援付きエディタとしての基本機能
- チャット、編集、エージェントの違い
- ファイルコンテキストの渡し方
- GitHubとの連携イメージ

### 2. エージェントに任せやすい仕事

- 調査、要約、分類
- ドキュメント作成
- 既存資料の改善
- テンプレート作成
- 軽微なコード修正
- テスト、検証、レビュー補助

### 3. 依頼文の作り方

依頼には以下を含める。

```text
目的:
背景:
対象ファイル:
期待する成果物:
制約:
確認方法:
```

### 4. GitHub Issueで管理する

- 依頼内容をIssueにする
- ラベルでカテゴリを分ける
- 成果物の確認条件を明記する
- エージェントがPull Requestを作成する流れを説明する

### 5. Pull Requestでレビューする

- 変更差分を見る
- 自分の意図と合っているか確認する
- テスト結果やチェックリストを見る
- 修正依頼をコメントする

### 6. 実務への落とし込み

- マーケティング資料の改善
- 営業メールの雛形化
- FAQ作成
- キャンペーン案の整理
- レポートの要約
- 社内ナレッジ整備

## デモ例

1. Issueテンプレートから「キャンペーンLP改善案の整理」を作成する
2. エージェントに依頼文を渡す
3. エージェントがドキュメントを追加する
4. Pull Requestで差分を確認する
5. レビューコメントを元に修正する

## 講師メモ

- AIは判断者ではなく作業者として扱う
- 依頼が曖昧なら成果物も曖昧になる
- 成果物の確認責任は人間に残る
- 小さな成功例を積み重ねることが定着への近道である
