# エージェント依頼テンプレート

Cursorエージェントへ依頼するときの基本フォーマットです。

曖昧な依頼は、曖昧な成果物を生む。現実を見るんだ。相手がAIでも、任務説明は必要だ。

## 基本テンプレート

```markdown
## 目的

何を達成したいのかを書いてください。

## 背景

なぜこの作業が必要なのかを書いてください。

## 依頼内容

エージェントに実行してほしい作業を箇条書きで書いてください。

## 成果物

完了時に何ができていればよいかを書いてください。

## 制約

守るべきルール、使ってはいけない表現、変更してはいけない範囲を書いてください。

## 参考情報

関連URL、既存資料、過去のIssue、ブランドガイドラインなどを書いてください。

## 確認方法

人間が何を見て完了判断するのかを書いてください。
```

## 例: 記事構成案の作成

```markdown
## 目的

新サービス紹介記事の構成案を作りたい。

## 背景

営業資料だけではサービスの価値が伝わりにくいため、Web記事として説明したい。

## 依頼内容

- 想定読者を3パターンに分ける
- 読者ごとの課題を整理する
- 記事タイトル案を5つ作る
- 見出し構成を作る
- CTA案を提案する

## 成果物

Markdown形式の記事構成案。

## 制約

- 誇大表現は避ける
- 専門用語には簡単な説明を添える
- まだ公開されていない機能は書かない

## 参考情報

- 既存営業資料
- 競合サービス比較表
- ブランドトーンガイド

## 確認方法

マーケティング担当者が、タイトル・構成・CTAをレビューする。
```

## 例: GitHub Issue用の短い依頼

```markdown
## 目的

社内向けCursor講義で使うサンプルIssueを作る。

## 依頼内容

- マーケティング業務で使えるAIエージェント依頼例を10件作る
- 難易度を beginner / intermediate / advanced に分ける
- 各依頼に期待成果物とレビュー観点を付ける

## 成果物

`docs/sample-agent-tasks.md`

## 確認方法

講義中に受講者が自分の業務へ置き換えられる内容になっているか確認する。
```

## 依頼を強くするチェックリスト

- [ ] 目的が一文で説明できる
- [ ] 成果物の形式が明確である
- [ ] 変更してよい範囲が明確である
- [ ] 参考情報がまとまっている
- [ ] 完了判断の基準がある
- [ ] 人間が判断すべき点を残している
